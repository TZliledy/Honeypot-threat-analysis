# Honeypot Deployment & Threat Analysis

## Overview
This project documents the deployment and analysis of a honeypot environment used to monitor suspicious network activity, unauthorized access attempts, and common attacker behavior. The goal of this project was to better understand defensive cybersecurity concepts, log analysis, and threat detection.

## Tools & Technologies
- Honeypot environment
- Linux
- VMware
- Network monitoring
- Log analysis
- Cybersecurity lab environment

## Project Objectives
- Deploy a honeypot in a controlled lab environment
- Monitor incoming connection attempts
- Analyze logs for suspicious behavior
- Identify common attack patterns
- Strengthen understanding of threat detection and defensive security

## What I Did
- Configured a honeypot to capture suspicious network activity
- Monitored login attempts and connection activity
- Reviewed logs to identify possible unauthorized access attempts
- Documented observed behavior and key findings
- Practiced defensive security analysis in a lab environment

## Key Findings
- Honeypots can be useful for observing attacker behavior in a controlled environment
- Log analysis is important for identifying suspicious activity
- Repeated connection attempts may indicate scanning, brute-force attempts, or automated attacks
- Defensive monitoring helps improve incident response awareness

## Sample Log Analysis

Example of activity observed in the honeypot environment:

- Multiple connection attempts from external IP addresses  
- Repeated login attempts indicating possible brute-force behavior  
- Unrecognized access attempts targeting open ports  

This type of activity highlights how attackers scan and attempt to gain unauthorized access to systems.
